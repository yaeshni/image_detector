Spot the Fake Photo — Real vs Screen-Recapture Classifier
Given one image, output a score from 0 (real photo) to 1 (photo of a screen).
python predict.py some_image.jpg  →  0.93
Requirements
pip install tensorflow opencv-python scikit-learn joblib scipy scikit-image
Python 3.8+ recommended. TensorFlow 2.x.
Project Structure
  predict.py       ← submission script (run this)   train.py         ← training pipeline   model.pkl        ← saved classifier (generated by train.py)   data/     real/          ← training: real photos     screen/        ← training: screen recaptures     val/real/      ← validation: real photos     val/screen/    ← validation: screen recaptures
How to Run
Step 1 — Train the model (only needed once):
  python train.py
This will print per-image progress, validation accuracy, any misclassified images, and save model.pkl.
Step 2 — Predict on a new image:
  python predict.py path/to/image.jpg
Output: a single number between 0 and 1. Values above ~0.5 indicate a screen recapture; values below indicate a real photo.
How It Works
MobileNetV2 (pretrained on ImageNet, weights frozen) is used as a feature extractor. Each image is resized to 224×224, passed through the network, and a 1280-dimensional embedding is extracted from the global average pooling layer. A logistic regression classifier trained on these embeddings decides real vs. screen.
The CNN is not fine-tuned — with ~70 training images, fine-tuning would overfit. The frozen embedding approach generalizes much better at this data scale.
Accuracy
●	Validation accuracy: 95% (19/20 held-out images correct)
●	1 misclassified image: a screen photo predicted as real
●	Note: small validation set (20 images); true accuracy may vary on a larger held-out set
Latency & Cost
●	Per-image inference: ~40–80 ms on laptop CPU (model loaded once)
●	On-device (TFLite): ~$0/image, sub-50ms on mid-range phones
●	Cloud CPU: ~$0.001–0.005 per 1,000 images
Limitations
●	Small training set (~80 images) — more data, especially more screen types (TV, tablet, OLED), would improve robustness.
●	Handcrafted classical features (FFT moiré, LBP, color variance) were explored but did not separate the classes reliably on phone-camera JPEGs. Transfer learning was significantly more effective.
●	Not yet converted to TFLite for production on-device deployment.
